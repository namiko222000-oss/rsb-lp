# LP公開・更新マニュアル（GitHub × Cloudflare Pages編）

このマニュアルは、作成したLPのデータを「インターネット上に公開」し、「最新の状態に更新」するための手順書です。決済（商用利用）に対応した安全な構成になっています。

---

## 【第1部】最初の1回だけ！LPをネット上に公開する手順
（※すでにGitHubとCloudflareのアカウント登録が終わっている前提です）

### Step 1: パソコンのデータをGitHub（ネット上の倉庫）に入れる
1. パソコンで「GitHub Desktop」アプリを開きます。
2. LPのデータが入っているフォルダ（今回の場合は `08RSB LP`）を、GitHub Desktopの画面の真ん中にドラッグ＆ドロップして登録します。
3. 画面左下の「Summary」という入力欄に「最初のコミット」など適当な題名を書き、青い **「Commit to main」** ボタンをクリックします。
4. 画面上部にある **「Publish repository」** ボタンをクリックします。（※この時、「Keep this code private（非公開にする）」のチェックは入れたままでOKです）
5. これで、パソコン内のデータがネット上の倉庫（GitHub）に安全にコピーされました。

### Step 2: Cloudflare（公開用サーバー）と連携させる
1. ブラウザで [Cloudflareの管理画面](https://dash.cloudflare.com/) にログインします。
2. 左隣のメニューから「Workers & Pages」を開き、青い **「アプリケーションの作成」** ボタンを押します。
3. **「Pages」** というタブを選択し、**「Git に接続」** ボタンをクリックします。
4. ご自身のGitHubアカウントを承認し、先ほどアップロードした「リポジトリ（08RSB LPなど）」を選択して「セットアップの開始」を押します。
5. 難しい設定はそのままスルーして、一番下の **「保存してデプロイ」** を押します。
6. これで完了です！数分待つと、Cloudflareから発行されたURL（例: `https://rsb-xxx.pages.dev`）で、全世界からLPが見れるようになります。

## 【第2部】普段の作業！LPを編集してネット上を更新する手順

AI（私）が指示を受けてLPを修正したり、夏緒さんご自身がテキストを書き換えたりしたあと、それを「ネット上の実際のページ」に反映させる手順です。
（※第1部の設定が終わっていれば、**以下の3ステップだけで完全自動でネット上が更新されます**。Cloudflareを開く必要すらありません）

### ネット更新の手順（毎回これをやります）

1. **GitHub Desktopを開く**
   パソコンで「GitHub Desktop」アプリを起動します。
   （※自動的に、LPの変更されたファイル一覧が左側に表示されています）

2. **変更を確定する（Commit - コミット）**
   画面左下の「Summary（概要）」という入力欄に、「キャッチコピー修正」「価格変更」など、何を変えたかわかる簡単なメモを入れます。
   メモを入れたら、そのすぐ下にある **「Commit to main」** という青いボタンをクリックします。

3. **ネットに送信する（Push - プッシュ）**
   画面上部にある **「Push origin」** というボタン（上向きの矢印⬆️アイコン）をクリックします。

**【以上で完了です！】**
たったこれだけです。Pushの矢印ボタンを押してから **約1〜2分後**、お手元のスマホなどでLPのURLにアクセスし、画面を再読み込み（リロード）してみてください。自動的に最新の状態に切り替わっています。

もしGitHub Desktopで変なエラーメッセージが出たり、「Pushできない！」という状況になったら、すぐにAI（私）に聞いてください。解決策をナビゲートします。
